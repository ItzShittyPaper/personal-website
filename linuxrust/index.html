Linux gets Rusty: My Response

all i can say is: Why?

Oxidization

first, we need to look at the bigger picture.

Rust is a new multi-paradigm programming language now commonly used for desktop applications and utilities. It’s biggest “pros” are efficiency and memory safety. For context i will just add that Rust is popular among the software development community. Especially the more “Leftist” side of it.

Linux is implementing Rust simply because there aren’t as many C developers as one would think. Most senior developers working on low-level code are now retired - or dead. Every year people are stepping down from the kernel and moving on with their lives. Basically - C is a dead language that zoomers are too afraid to learn, and the supply of people programming in it is quickly depleting.

Blahaj, Programmer Socks, and Rust

Let’s debunk Rust into oblivion, shall we?

  1. Efficiency

Rust is more efficient than its biggest competitor, C++ - at least in most cases. It’s mostly because unlike C++ it doesn’t have garbage collection. Instead it uses the concepts of ownership and borrowing. Ownership basically is a system in which memory is automatically freed when a variable is not needed. This generally saves memory space, but increases CPU usage and can lead to some other problems. In C, developer does this manually with the free() function. Even if this is not needed in most cases.

Basically, C is much faster, and modern C compilers can compile memory leak-less code without much trouble.

  1. Memory Safety

Rust fanboys think that it’s 100% memory safe. And it’s a lie. Rust in no way guarantees memory safety. The only difference between “unsafe” languages like C and rust, is that the Rust compiler flags unsafe code as errors that you need to correct.

The kernel of C

Still, EXPERIMENTAL Rust support is being added to the upcoming linux 6.1 release. Most distributions will include Rust maybe in a year or two.

Some of them will probably disable Rust support for some time after the 6.1 release, as it is currently optional.

If It Moves, Compile It

My biggest critic of Rust in the kernel is the time it takes to compile. Another language in the build toolchain will basically make the kernel twice as heavy to run and build.

Linux was historically a very lightweight kernel, despite of it’s monolithic nature. Now it is becoming more and more bloated.

I mentioned this in my previous blog post, but corporate sponsors are one of the main reasons things like Rust get included in the kernel. In a few years many people with low-end hardware will probably need to change their setup because of this change (and many others).

Basically, sticking to facts: soon compiling the kernel will take around 8 hours on mid-range hardware. imagine using gentoo with these compile times!

But Should You?

for now i’ll try use linux even with all of this. but in the future? i will probably switch to BSD. even without some games, i should be fine with the linux binary compatibility. idk.
