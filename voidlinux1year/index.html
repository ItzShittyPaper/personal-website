Void Linux: a Year long review

tl;dr: don’t use it

Void Linux is one of those distros that you should probably setup on a secondary, low-powered machine that doesn’t run modern Windows very well. That is the best use for a “DIY” distro, after all. You set it up with your computer in mind to get maximum performance available, and then sit back and use it sometimes for watching movies, torrenting and other stuff like that - nothing serious. After using this distribution for a year, i am pretty confident that this is how void is being developed and tested, and that most people don’t run it on their main machines. The height of the thinkpad meme (and i’m saying this as a person that actually bought a thinkpad, even though i don’t really need one and i use it only on trips occasionally, maybe in bed). Too bad i decided to completely disregard it and install it on my main production machine i’m literally writing this on. Now i’m on openSUSE Tumbleweed, after over a year of using Void on this PC. Today i want to share my experience with Void, and highlight where it’s really good, and where it shits itself - which is arguably more important.

The Good!

Let’s start with what Void Linux does right, just to add some pint of optimism to this rant about the distro. At first i should mention that void is really fast. Like it’s one of the fastest distros i have ever used, better than even most of my arch setups. While it’s not optimized like gentoo, it’s good enough for most people. The kernel uses a reasonable configuration, but you might have problems with the realtek wi-fi drivers. I don’t know honestly, generally hardware support isn’t bad at all. The best thing about Void is probably the guided installer, which automatically installs the base OS so you can configure it. It kinda resembles systems like Slackware or even MS-DOS, making for a good compromise between modern graphical installation programs and the manual setup of Arch and Gentoo. Also it’s fully in the command line, working even in secure shell and other environments where graphics aren’t a priority. Or to put it in a different way, the installer will still work without graphics drivers, it won’t crash on you. You obviously can also use your system for recovery, but it’s not ideal. I’d rather use something like debian or even Ubuntu. XBPS is also a pretty good packaging system, it’s one of the more lightwieght ones, and it’s better than stuff you get in Arch and Debian for example. The only thing i’d want is a single feature from apt, the autoremove which automatically removes unnecessary dependencies. You CAN remove a package with all it’s redundant dependencies, but if you forget to uninstall it with -R, you’re screwed, the dependencies stay on your system and you can easily forget about them. Obviously i should state the obvious. This is a do it yourself distro, you can pick yourself every element of your system, like the desktop environment and most software. It’s generally a good balance between hellholes like gentoo and the ease of use (and rigidity) of stuff like Ubuntu or Manjaro. But, that’s where the good parts end i think, the rest is really bad and infuriating, knowing how good this distribution could be.

The Bad

Now let’s talk about the bad, infuriating, and truly crap parts of Void Linux. It can be really bad sometimes, so let’s just get into it. First, runit is really confusing at parts, it’s nearly undocumented, so you can do something wrong and never recieve an error. It’s literally ed levels of feedback, and you often have to rely on your intuition with runit. The same goes for some features of the XBPS package manager. Also the packaging of some programs like GNOME is weird, providing outdated software and requiring additional setup, which is something that you’re never told about. The GDM package is really tricky for example. Another big problem are the repositories, which very often don’t have what you want, for example most proprietary apps like Discord, or the remote desktop tool Parsec. For some reason steam is in the main repos though. I don’t know why that’s the case, but i suspect one of the maintainers really depends on steam, or they just like valve. You actually CAN install discord through the void-packages repository, where you can use a tool called xbps-src, that is similar to the AUR (while not being entirely user contributed, it’s a very small selection of additional packages from my experience) to install it. Apparently you can also use it to install parsec, but i didn’t have any luck with it. Generally it’s really confusing. These packages break often and after some time i had to just use the provided tarball for discord / use the browser, which is a bummer. I wouldn’t use discord in the first place but i need it to contact my friends, so you know how it is. If you want more advanced stuff like ROCm, just forget about it. I don’t think you can get blender to use your GPU at all, no matter the vendor. It’s not possible on AMD, and it’s probably shitty on NVIDIA graphics cards as well. What a shame. Another thing is also flatpaks were broken on my system, so replacing the often breaking packages from void with their flatpak counterparts (previously mentioned discord, parsec) didn’t work, it just made the issue much worse. Flatpak packages on void often had problems with theming, detection of the microphone, file picker issues etc. Wouldn’t recommend. I haven’t tried snaps, but they have their own problems so i don’t think i should try them anytime soon. Generally the OS can gather bitrot pretty quickly without maintenance due to broken packages etc. It’s the only rolling release that made me actually cautious of updating the OS after a while. In most systems like arch i could update after 2 months of not upgrading anything and it would work reasonably well, sadly that isn’t the case with Void. I often would have to switch desktop environments because of breakages, and while at the start i used GNOME 40 / 41 (i don’t remember exactly), at the end i used MATE, and used almost everything from KDE to window managers and XFCE. Void acts like Windows Millenium in a way, just gathering bugs quicker than it should. I need to also criticize the Void team a little bit. Their documentation is awful and it’s nearly empty. It doesn’t contain any information regarding actually installing packages an such, only general stuff and a quick XBPS tutorial. It’s a far cry from resources like the arch or gentoo wiki, which are much more helpful despite not being made with void linux in mind. The worst thing is that previously there was a Void Linux wiki, that was probably much better and it actually did contain useful information. Sadly they closed it i think. Now all you get is a simple “Void Linux Docs” page that contains as much info as an Apple quick start guide.

The Conclusion

To wrap up my rant, i’m going to say that void is a really fast distro and it’s generally usable for those who want a Linux for their secondary machine (a laptop, old pc etc.), it’s unfortunately not suitable for a main production system, and it lacks many important packages that people use every day on other distributions. Runit - while fast - lacks important info and can be confusing. I wouldn’t recommend it over something like openrc that is almost just as fast with parallelization enabled. If you want a “do it yourself” linux just get arch or something else, void’s not worth your time until they fix these problems, and a few more. Sorry for being a downer i guess.
